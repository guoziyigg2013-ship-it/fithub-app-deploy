# FitHub App Store 与微信小程序上线清单

## 目标

把当前 FitHub 从“可演示网页试用版”推进到“可审核、可内测、可运营”的 iOS App 与微信小程序版本。

优先原则：

- 先保证账号和用户数据长期稳定
- 再补审核合规和内容安全
- 最后做原生能力、商业化和正式上架

## 一、上线前必须补齐

1. 账号体系正式化
   - 手机号验证码作为主账号
   - 一个手机号只能对应一个主账号
   - 健身爱好者、教练、健身房三种身份都挂在同一主账号下
   - 预留微信登录和 Sign in with Apple 绑定
   - 支持退出登录、跨设备登录、身份切换和账号注销

2. 数据生产化
   - 后端迁到更稳定的国内云服务
   - 数据库、对象存储和 CDN 使用生产配置
   - 用户、关注、动态、私信、预约、打卡都要有备份与恢复方案

3. 合规与审核
   - 用户协议
   - 隐私政策
   - 权限说明
   - 账号注销入口
   - 数据删除申请入口
   - 未成年人保护说明

4. UGC 内容安全
   - 动态、评论、图片、视频、私信内容审核
   - 举报、拉黑、删除、封禁
   - 管理员后台处理用户与内容

5. 定位与健康权限
   - GPS 轨迹权限说明
   - 附近推荐权限说明
   - HealthKit / 手表 / 智能秤数据授权说明
   - 不授权时仍能使用基础功能

6. 微信小程序准备
   - 小程序账号与主体
   - 已备案 HTTPS 合法域名
   - 小程序隐私保护指引
   - 服务类目确认
   - 微信内容安全接口
   - 微信登录与手机号能力

7. iOS 原生化准备
   - 不是简单网页套壳
   - 原生登录与权限弹窗
   - 相机、相册、定位、通知、HealthKit
   - TestFlight 内测账号与审核演示账号

8. 运营后台
   - 用户管理
   - 教练/健身房入驻审核
   - 动态与评论审核
   - 订单与预约查看
   - 举报处理

## 二、建议执行顺序

### 第 1 步：账号体系正式化

目标：

- 把手机号作为主账号，不再让不同身份变成互相割裂的账号
- 同一手机号可以拥有训练者、教练、健身房三种身份
- 预留微信和 Apple 登录绑定字段，后续接原生 App 与小程序时不用重做数据结构

当前已落地：

- 主账号记录新增正式身份绑定模型：
  - `primaryProvider`
  - `identityProviders`
  - `authVersion`
- 手机号会作为已验证登录提供方写入账号
- `managedAccounts` 会返回：
  - 手机号脱敏展示
  - 已绑定登录方式
  - 待绑定的微信和 Apple
  - 当前账号拥有的三种身份
- 新增 API 回归测试，确认同手机号注册训练者、教练、健身房后仍共用同一个主账号

### 第 2 步：国内生产存储与后端

目标：

- 明确国内云服务方案
- 配生产数据库
- 配对象存储和 CDN
- 设计每日备份和恢复演练

当前已落地：

- 新增 `/api/storage/status` 存储健康诊断接口，发布后可直接确认后端是否仍在使用 Supabase 持久化
- 发布后 smoke 会检查存储模式，避免误把生产服务跑成本地 JSON
- Supabase 状态备份增加保留窗口，默认保留最近 96 个小时备份，避免备份行无限增长
- 保留手机号恢复行和主状态备份两层保护，新注册账号、关注、动态、私信、预约、打卡仍会被纳入恢复范围

下一步：

- 选择中国大陆长期后端供应商：腾讯云 / 阿里云 / 华为云三选一
- 把当前 Python API 迁到国内云主机或容器服务
- 把前端固定域升级为已备案自有域名
- 配置对象存储 CDN 加速真实用户图片和视频

### 第 3 步：内容安全与举报后台

目标：

- 先做不影响用户体验的最小内容安全闭环
- 用户可以举报动态
- 动态、评论、私信的可疑文本会进入审核队列
- 管理员可以通过受保护接口查看待处理举报和审核项

当前已落地：

- 新增内容审核基础模型：
  - `reports`
  - `moderationQueue`
  - `adminActions`
- 动态发布、评论、私信会进行轻量关键词扫描，可疑内容不中断发布，但会进入待审核队列
- 探索动态卡片新增“举报”入口，举报会持久化并进入审核队列
- 新增受 token 保护的管理员接口：
  - `GET /api/admin/moderation`
  - `POST /api/admin/moderation/resolve`
- 新增 API 回归测试，覆盖风险内容入队、用户举报持久化、管理员接口权限保护

下一步：

- 把图片、视频接入云厂商内容安全审核
- 增加拉黑、隐藏、删除、封禁动作
- 做一个真正可视化的运营后台页面

### 第 4 步：微信小程序 MVP

待做：

- 小程序项目结构
- 微信登录
- 首页、探索、动态、预约、我的
- 图片/视频上传
- 内容安全接口

### 第 5 步：iOS TestFlight MVP

待做：

- 原生壳接入当前后端
- 原生权限、定位、相册、HealthKit
- TestFlight 内测
- App Store 审核材料

### 第 6 步：正式上架与运营

待做：

- 隐私政策与用户协议上线
- 审核账号准备
- 商业化模块确认
- 客服与反馈机制
